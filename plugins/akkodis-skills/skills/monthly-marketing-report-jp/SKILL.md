# 月次マーケティングトレンドレポート生成スキル

## スキルの目的

AKKODiSコンサルティング株式会社(https://www.akkodis.com/ja)のマーケティング部が、月次の経営会議で使用するマーケティングトレンドレポートを、一貫した品質・フォーマットで生成する。経営層向けに、エンジニア派遣・コンサルティング・SIer業界の最新動向を要約する。

## 起動時の確認事項

スキル起動時、以下をユーザーに確認する(既に指定されていればスキップ)。

1. **対象年月**(例: 2026年3月) — 必須
2. **生成するパート** — 以下から選択。**推奨はパート単位の個別実行**（1回1パート）。全パートは合計66トピック・検証込み100回超のツールコールとなり、単一応答では途中打ち切り／レート制限／件数不足を招くため。未指定の場合はどのパートかを確認する（対象年月と併せて一度に確認）。
   - Part① マーケット概況・トレンドのみ（推奨単位）
   - Part② 競合動向のみ（推奨単位）
   - Part③ 産業動向のみ（推奨単位）
   - 全パート（非推奨。指定時は分割実行を提案し、続行指示があれば Part①→②→③ の順に生成）
3. **出力形式** — デフォルトは**チャット内テキスト**。ユーザーが「docx」「Wordで」「ファイルで」等を明示した場合のみ、`docx` skillと連携して.docxファイル化する。

## 共通の前提条件(全パート共通で自動注入)

```
・レポートの目的: 月次の経営会議資料にマーケティングトレンドレポートを掲載することで、マーケティング部としての存在価値を示す
・レポートターゲット: 経営層
・業界ターゲット: エンジニア派遣、コンサルティング、SIer
・サービスターゲット: エンジニア派遣、コンサルティング事業、開発請負
・自社: AKKODiSコンサルティング株式会社(https://www.akkodis.com/ja)
・出力言語: 日本語
```

## 絶対遵守ルール(全パート共通)

`rules/anti_hallucination.md` および `rules/source_policy.md` を**必ず参照**してから生成を開始すること。要点:

1. **対象月限定（拡張定義）**: 採用可とするのは、`rules/source_policy.md` の「対象月の情報」の定義に従い、(A)一次情報の発表日／公開日が対象月内、または(B)対象月内に発生・発表された出来事を対象月内公開の記事が報じたもの、のいずれか。前月分の集計でも**当月発表**なら採用可。汎用記事の「更新日だけ対象月」や前月以前事象のまとめ直しは除外。
2. **参照元URL必須**: 全トピックに必ずURLを1つ以上記載する。
3. **ハルシネーション厳禁**: 数値・社名・サービス名・日付は参照元で確認できたもののみ記載。不確実な情報は「情報なし」と明記し、推測で埋めない。
4. **具体性**: サービス名・数値・固有名詞を必ず含める。抽象的な表現だけで済ませない。
5. **IR情報除外(Part②のみ)**: 決算短信・月次報告・有価証券報告書などのIR情報は使用しない。プレスリリース・お知らせを優先する。
6. **検索は2段階で行う（記憶に頼らない）**:
   - 第1段（母数確保・recall優先）: `web_search` で候補を最大化する。**クエリに月指定（例「2026年4月」）を入れない**——月をクエリに入れると一次情報がヒットせず母数が激減するため。年単位（例「2026」）までは可。`site:`・PR TIMES・業界メディア名などの情報源指定クエリを併用する。
   - 第2段（対象月の確定・precision担保）: 第1段で得た候補URLを `web_fetch` し、`rules/anti_hallucination.md` の日付ゲート(A)(B)で対象月を確認する。**対象月の要件はクエリではなく検証で担保する。**
7. **検索回数の下限（フロア）を満たす**: 各パートの `references/` に定めた最低検索回数とクエリ・マトリクスを満たすまで探索を打ち切らない。
8. **早期離脱の禁止（件数不足宣言の条件）**: 「件数不足」を宣言してよいのは、(a)当該パートのクエリ・マトリクスを全角度試し、(b)最低検索回数を満たし、(c)情報源指定クエリ（site:/PR TIMES/業界メディア）も試した後に限る。それ以前の離脱は禁止。ただし件数を埋めるための捏造（ハルシネーション）は引き続き厳禁——正直な件数報告は残す。

## パート別の処理

要求されたパートに応じて、対応する `references/` 配下のファイルを読み込み、その指示に従って生成する。

- **Part①** → `references/part1_market_trend.md`
- **Part②** → `references/part2_competitors.md`
- **Part③** → `references/part3_industries.md`

全パート要求の場合は、Part① → Part② → Part③ の順に生成する。

## 出力フォーマット

`templates/output_format.md` を参照し、統一されたフォーマットで出力する。

- [ ] 各記事について、(A)一次情報の発表日／公開日、または(B)出来事の発生日＋記事公開日のいずれかが対象月内であることを `web_fetch` で本文から確認してから採用しているか（検索スニペットの日付のみでの判断は不可）

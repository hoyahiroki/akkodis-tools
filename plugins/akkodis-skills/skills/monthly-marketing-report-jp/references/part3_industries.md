# Part③ 産業動向(エンジニア派遣ニーズ)

## 生成指示

指定された対象月(例: 2026年3月)にリリースされたインターネット上の情報を基に、以下の5産業それぞれについて、エンジニア派遣ニーズに関連するトピックを各5件ずつ、参照元URL付きで生成する。

## 対象産業(5つ)

1. **自動車・運輸産業**
2. **航空宇宙・防衛産業**
3. **ICT産業**
4. **ヘルスケア産業**
5. **クリーンテクノロジー産業**

## 検索戦略（2段階）

### 大原則
対象月の要件は**クエリではなく検証（web_fetch）で担保する**。**第1段のクエリに月指定（例「2026年4月」）を入れない**（母数が激減するため）。年単位（例「2026」）までは可。

### 第1段: 母数確保（recall優先）
各産業ごとに、以下のマトリクスを**最低6角度以上**投げる（[産業名]を各産業に、[YYYY]を対象年に置換。月は入れない）:
- 「[産業名] エンジニア 人材不足」
- 「[産業名] DX 人材 需要 [YYYY]」
- 「[産業名] 技術者 採用 拡大」
- 「[産業名] 新規プロジェクト 発表」
- 「[産業名] 設備投資 [YYYY]」（間接的に技術者需要を示唆）
- 「[産業名] 工場 拠点 新設」
- 情報源指定: 「site:prtimes.jp [産業名] エンジニア」「[産業名] 業界 ニュース」

### 第2段: 対象月の確定（precision担保）
第1段で得た候補URLを `web_fetch` し、`anti_hallucination.md` の日付ゲート(A)(B)で対象月を確認する。スニペット日付のみでの採用は不可。

### 検索回数フロア（厳守）
- **各産業 最低3回、本パート全体で最低15回**の web_search を行う。これに達する前に、その産業を「件数不足」と断定してはならない。
- フロア・全クエリ角度・情報源指定クエリを試したうえでなお不足する場合のみ、下記「厳守事項」の件数不足報告に従う。

## トピックの選定基準

以下のような情報をエンジニア派遣ニーズの根拠として採用する:
- 産業内で進行中/新規発表されたDX・技術革新プロジェクト
- 業界全体の人材不足・採用動向に関する調査やコメント
- 大型投資・設備投資・R&D強化の発表(技術者需要を示唆)
- 業界レギュレーション変更に伴う人材ニーズ
- 新規工場・拠点設立(技術者の配置需要)

## 出力内容

各産業ごとに以下のフォーマットで5件生成:

```
■ [産業名]
1. [トピック要約 2〜3行、エンジニア派遣ニーズにどう結びつくかを含めて記述。具体的な企業名・プロジェクト名・数値などを盛り込む]
   参照元: [URL]
2. [同上]
   …
5. [同上]
```

## 厳守事項

- **当月リリースの情報のみ**使用
- 参照元URLは**全トピックで必須**
- 5件に満たない場合は、上記フロア（各産業最低3回・全角度・情報源指定）を試し切ったうえで、無理に作らず取得できた件数だけ出力し不足を明記（フロア未達での早期離脱は禁止）
- 数値・企業名・プロジェクト名は参照元で確認できたもののみ使用。推測厳禁。

# 表記ルール準拠スキル

同梱の `表記ルール.md`（五十音順の表記辞書）に厳密準拠して、日本語原稿の表記を判定・校正する。

## 最重要原則

- **辞書に載っている語は、必ず辞書の表記に従う。** 一般的な感覚より辞書を優先する。
- 辞書に**載っていない語**は勝手に直さない。触る場合は「辞書外・一般基準による推定」と明示する。
- **辞書を全文コンテキストに読み込まない。** 約19,000行あるため、対象語だけを `grep` で引く（下記手順）。

## 辞書の引き方（必須手順）

エントリ書式は `-   **よみ**: 正しい表記（×誤表記）／…`。**見出しキーは語の「よみ（平仮名）」**。

1. **よみが分かる語** → よみで引く
   ```bash
   grep -nA2 "\*\*あかり\*\*" 表記ルール.md
   ```
2. **よみが不確かな語** → よみの部分一致、または漢字そのものを本文値から引く
   ```bash
   grep -n "あがる" 表記ルール.md      # よみ部分一致
   grep -n "灯り" 表記ルール.md         # 漢字（誤表記側にヒットすることがある）
   ```
3. 複数候補が出たら、エントリ本文の使い分け説明（`（ ）` 内の基準、`※`、`>` の同音異義語列挙）を読んで文脈で選ぶ。

`-nA2` で見出し＋続く2行（`>` の複数表記行や `※` 注記）まで取れる。長いエントリは範囲を広げる。

## 記号の意味と処理

| 記号 | 意味 | 処理 |
|------|------|------|
| `×` | 誤った・非推奨表記 | **絶対に使わない**。正しい表記に置換 |
| `⊗` | 差別語・不快語・使用不可語 | **使用禁止**。代替語に言い換える（言い換え候補を提示） |
| `△` | ルビが必要／使用注意の漢字 | ルビを付けるか、平易な語に開く |
| `※` | 補足・注意事項 | 記載内容に従う |
| `➡` | 関連項目への参照 | 参照先エントリも引いて確認 |
| `／` | 文脈による使い分け | エントリの `（説明）` に従って選択。自動で1つに決めず、判断根拠を示す |

判断に迷う場合の優先順位：①辞書の記載表記を最優先 → ②漢字/平仮名/カタカナの選択も辞書の指示に従う → ③辞書に無い語は一般的な新聞・放送メディアの表記基準に準じる（＝推定である旨を明示）。

## 2つの使い方

### A. 単語照会
「◯◯は漢字か平仮名か」等。対象語を `grep` で引き、正しい表記＋根拠（記号・使い分け説明）を返す。辞書に無ければ「辞書外」と明示。

### B. 原稿スキャン／校正
原稿テキストを受け取り、辞書に該当する語を洗い出して違反を検出する。

- 出力は【修正後本文】＋【修正一覧（語・元表記→修正表記・根拠記号・辞書の該当説明）】の2部構成。
- `×`・`⊗` は必ず修正対象。`／`（使い分け）は文脈判定し、判断が割れる箇所は「要確認」として残す。
- **辞書に無い語は改変しない。** 「辞書に記載なし」の語まで直すと誤修正になるため、勝手に触らない。
- ハルシネーション厳禁。辞書に無い根拠を「辞書にある」と偽らない。grep結果に基づいてのみ判定する。

## 対象外

- 校閲（事実・固有名詞の正誤確認）
- 推敲（文体・語順・冗長性のリライト）
- 辞書に載っていない語の表記判断を、辞書準拠であるかのように断定すること

これらを求められたら、本スキルの範囲外である旨を先に述べる。

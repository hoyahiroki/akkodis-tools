# Web掲載用 人物写真 撮影要件（自動トーン統一の前提条件）

目的: 撮影回・拠点が違っても自動トーン統一が安定して効く素材を確保する。
原則: **色被り・露出は「撮影時に正しく」が最優先。後処理は微調整に留める。**
後処理で直せること=軽い色被り/黒白点/彩度。直せないこと=**強い色被り下の顔色**、**逆光で暗い顔だけの持ち上げ**、**白飛びの復元**。

## 必須要件
1. **ホワイトバランス**: 各シーン冒頭にグレーカード(18%)を1カット。AWB任せにせずカスタム/マニュアルWB。混合光源(窓光+タングステン等)を避ける。
2. **顔の露出**: 顔基準で露出を決める。明るい背景(窓・白壁)の前は +0.7〜1EV、またはレフ/ストロボで顔を起こす。
3. **階調**: ゼブラ/ハイライト警告で白シャツ・白壁の白飛びを回避。黒つぶれも避ける。
4. **光源**: 金色間接照明・ネオン・夕日直射「のみ」の強い色の環境光を避ける。演出で使う場合は顔にデイライト光を足す。
5. **背景**: 大面積の原色(イベントバナー等)は人物の自動補正を阻害するため、可能なら人物比率を上げる。
6. **同一ロケ内**: WB・露出をマニュアル固定し、カット間で変えない。

## 納品仕様
- RAW+JPEG（最低でもRAW）。長辺3000px以上、sRGB、シャープネス弱。
- 同一シーンのカットはWB/露出が揃っていること。

## 検収基準（自動チェック。範囲外は再調整/再撮影を依頼）
| 項目 | 合格 | 注意 | 不可 |
|---|---|---|---|
| WB色被り a（緑-赤） | \|a\|≤3 | ≤6 | >6 |
| WB色被り b（青-黄） | \|b\|≤5 | ≤10 | >10 |
| 黒の締まり（L下位1%） | ≤8 | ≤14 | >14 |
| 白飛び率（L>97の画素%） | ≤8% | ≤18% | >18% |
| 露出（L中央値） | 40–75 | 30–85 | 範囲外 |
| 彩度（平均chroma） | 8–20 | 5–26 | 範囲外 |

検収コマンド: `python3 scripts/color_match.py 納品フォルダ --check`

# resource-content 構造リファレンス（client-stories / 実ソース準拠）

AKKODiS `client-stories` 事例記事の `<div class="resource-content">` 配下に出力するHTMLの正本。**実際の公開 client-stories ページのSitecoreソースから確定したクラス・入れ子・空白の扱い**を最優先で踏襲する。ここに無いクラスは作らない。`akkodis-insights-html` とは画像・Index・話者・パスのクラスがすべて別物なので、insightsの構造を持ち込まない。

**基本構成が「正」**：全要素がそろった標準的な事例記事の構成（§2）を基準テンプレートとし、実際のWord・レイアウト資料に応じて**要素が無ければ省く／異なれば合わせる**。**記事カテゴリは3種（アカデミー／ソリューション／コンサルティング）で、カテゴリは必須入力。カテゴリは主にCTAのDLボタンを決める**（§2, §6-3）。本文構造そのもの（Q&A/ナラティブ・Index/H3・導入ボックスの有無）はカテゴリと別にソースへ追従する。

**サンプル中の `《…》` はプレースホルダ**（実原稿の値に置換）。class名・タグ・定型文（CTAの固定文言など）はそのまま使う構造の一部。

## 1. 出力コントラクト

- ファイル1行目は厳密に `<div class="resource-content">`、末尾は対応する `</div>`。前後に空行・説明・コメント・Markdown・コードフェンス・BOM・引用符を置かない。
- 画像は必ず完全な `<img …>` 要素。`src` だけを裸テキストで出さない。
- `data-__outbound-listener-attached="true"` を **絶対に出力しない**（ブラウザの追跡JSが実行時に付与する属性）。
- **整形式HTMLを出す**。実ソースには `<p>` の閉じ忘れ・入れ子崩れ等のバグがあるが、**それは再現しない**（`<p>` は必ず開閉対応させる）。
- `?la=ja` の後ろに `&hash=…` を付けない（公開時にSitecoreが付与する。CMSに貼る入稿用HTMLには無い）。

## 2. 基本構成（正）と、ソースに応じた増減（最重要）

**基本構成＝全要素がそろった標準的な事例記事**。これを「正」とし、実際のWord・レイアウトに応じて要素を増減する。**記事カテゴリは3種（アカデミー／ソリューション／コンサルティング）で、カテゴリは必須入力**。カテゴリは主に**CTAのDLボタン**を決める（下表・§6-3）。**本文構造そのもの**（本文がQ&Aか地の文か、Index・H3・導入ボックスの有無）は、カテゴリとは別に個々の要素差としてWordの実態へ追従する。**カテゴリがプロンプトに無ければ生成前に質問して確認する**（推測で決めない）。

**基本構成（上から順）**：
1. MV画像
2. 導入ボックス（ロゴ＋メタ4項目＋プロジェクト概要）
3. CTAブロック（上, from=top）
4. Indexパネル（`mod-panel-anchor`）
5. 本文：`H2`＋`H3`＋`―質問`／話者ラベル付き段落、本文中の画像
6. CTAブロック（下, from=bottom）

**ソースに応じた増減**（実際のWord本文・レイアウト資料＝実ページ/PDF/画像/ユーザー指示 を読んで判断）：

| 要素 | 基本構成（正） | 記事によっては | 対応 |
|---|---|---|---|
| 本文形式 | 話者ラベル付きQ&A（`―質問`＋`姓さま：`／`AKKODiS 姓：`） | 三人称の地の文＋「」直接引用（話者ラベル無し） | Wordの本文形式に追従。ラベルが無ければ付けない（§6-6） |
| Indexパネル | あり（`抱えていた課題：`等の見出し＋箇条書きが原稿にある） | 無し（原稿に該当ブロックが無い） | 原稿に該当ブロックが無ければ省く |
| H3（サブ見出し） | あり | 無し（H2のみ） | 原稿にサブ見出しが無ければ省く |
| CTA下段/上段のDLボタン（右の白 `mod-btn-secondary`） | **カテゴリで決まる**：アカデミー・コンサルティング＝事例集（`akkodis-casestudies`） | ソリューション＝会社案内（`corporate-profile`） | 入力カテゴリに従う。未指定なら生成前に質問（左の黄色 `mod-btn-conversion`「法人向けご相談」は全カテゴリ共通） |
| キャプション氏名の強調 | `<strong>` | `<span class="mod-lyt-media__pointText">` | 基本は `<strong>`。実ページが `pointText` ならそれに合わせる（§6-7） |
| 段落マークアップ | 段落ごと別 `<p>` | 1セクションを単一 `<p>`＋`<br /><br />` | 基本は別 `<p>`。実ページが `<br /><br />` 連結ならそれに合わせる（§6-6） |
| 導入ボックス | あり（原則必須） | 稀に無し（イレギュラー） | ★下記の実行前確認 |

**★導入ボックスとメタ4項目は原則必須／実行前確認（必須）**：導入ボックス（ロゴ＋メタ4項目＋プロジェクト概要）は**原則すべての記事に付く前提**で扱う。メタ4項目（クライアント/担当部署/業界/支援内容）はWord本文に無いことが多いので**別途入力で受け取る**。**断片を生成する前に**、次のいずれかなら停止して確認する（このスキルでは推測で進めない）：
- メタ4項目がWordにも入力にも無い → **生成前に4項目を尋ねる**（空欄・捏造で埋めない）。
- 記事に導入ボックスが無いように見える（**稀なイレギュラー**）→ **生成前に「導入ボックス無しで正しいか」を確認する**。

## 3. resource-content の境界（何が中で何が外か）

公開ページのうち **`resource-content` に含めるのは下記のみ**。それ以外（記事タイトルH1・公開日・「サービス紹介」プロモ・「最新事例」関連記事・フッタ）は Sitecore テンプレート側で、**断片に含めない**。

```
resource-content の中身（上から順・基本構成）:
  MV画像
  導入ボックス（ロゴ＋メタ4項目＋プロジェクト概要）
  CTAブロック（上, from=top）
  Indexパネル ※原稿に該当ブロックが無ければ省く
  本文（H2 ＋ H3 ＋ 段落 ＋ 画像ブロック）
  CTAブロック（下, from=bottom）
```

- **記事タイトル（H1）と公開日は断片の外**（Sitecoreの別フィールド）。チャットでタイトルを別途提示する。
- **CTAブロックは resource-content の中**（上下2箇所とも）。定型文言のまま出力する（§6-3）。
- 「プロジェクト概要」（導入ボックス内）や、導入ボックスが無い記事のリード段落も **断片の中**（insightsはリードを外に出したが、client-storiesは中）。

## 4. 構成順（基本構成）

1. MV画像（§6-1）
2. 導入ボックス：ロゴ＋メタ4項目（クライアント/担当部署/業界/支援内容）＋プロジェクト概要（§6-2）
3. CTA（上, from=top）（§6-3）
4. Indexパネル（中核セクションのみ）（§6-4）　※原稿に該当ブロックが無ければ省く
5. 本文：各セクション `H2`＋`H3`＋`―質問`／話者段落、画像は本文の出現位置に挿入（§6-5〜6-7）
6. CTA（下, from=bottom）（§6-3）

**ソースに応じた増減**は §2 の表のとおり。導入ボックスが無い稀なイレギュラー記事では、MV → CTA(上) → リード段落（自由 `<p>`）→ 本文 → CTA(下) の順になる（実行前確認の上で）。本文・見出し・画像の順序は原稿どおり保持する。

## 5. 画像パスと命名

- パス（**全て小文字**。insightsの `Project/Akkodis` 大文字とは違う）:
  `/-/media/project/akkodis/akkodis/ja/images/client-stories/《slug》/img-《役割》.webp?la=ja`
- **`《slug》` はページURLの末尾セグメントと同一**（推測しない。URL未提供なら停止して要求）。例: `…/client-stories/《末尾セグメント》` → `《末尾セグメント》`。
- **ファイル名は役割から仮置きし、全て「要差し替え」として提示**。実ページの命名は不規則（`img-mv`／`img-mv-` のように拡張子前にダッシュが付く例が混在、位置・登場順を表す接尾辞が付く例もある）。規則を断定しない。**拡張子直前に不要な `-` は付けない**（`img-xxx.webp`。`img-xxx-.webp` にしない）。
- `&hash=` は付けない（§1）。
- **srcset を付けるのはロゴ画像のみ**（`@2x`）。本文・MV画像には付けない。

## 6. 各ブロックの正規HTML

### 6-1. MV画像（先頭）

```html
<div class="mod-lyt-media mod-lyt-media--mv">
<div class="mod-lyt-media__img">
<img src="/-/media/project/akkodis/akkodis/ja/images/client-stories/《slug》/img-mv.webp?la=ja" alt="《記事タイトル》" />
</div>
</div>
```

- `alt` は**記事タイトル全文**。

### 6-2. 導入ボックス（ロゴ＋メタ4項目＋プロジェクト概要）

```html
<div class="mod-box-introduction">
<div class="mod-box-introduction__head">
<p class="mod-box-introduction__media"><img src="/-/media/project/akkodis/akkodis/ja/images/client-stories/《slug》/img-logo.webp?la=ja" srcset="/-/media/project/akkodis/akkodis/ja/images/client-stories/《slug》/img-logo.webp?la=ja 1x, /-/media/project/akkodis/akkodis/ja/images/client-stories/《slug》/img-logo@2x.webp?la=ja 2x" alt="" /></p>
<dl class="mod-box-introduction__data">
<dt>クライアント</dt>
<dd>《会社名》</dd>
<dt>担当部署</dt>
<dd>《担当部署》</dd>
</dl>
<dl class="mod-box-introduction__data">
<dt>業界</dt>
<dd>《業界》</dd>
<dt>支援内容</dt>
<dd>《支援内容》</dd>
</dl>
</div>
<div class="mod-box-introduction__body">
<dl class="mod-box-introduction__data">
<dt>プロジェクト概要</dt>
<dd>
<p>《概要段落1》</p>
<p>《概要段落2》</p>
<p>《…取材した人物の列挙段落（「今回の取り組みについて、〇〇さま（…）に話を伺いました。」）まで含む》</p>
</dd>
</dl>
</div>
</div>
```

- **メタ4項目（クライアント／担当部署／業界／支援内容）は Word本文に無いことが多い**（実際の入稿原稿には表もテキストも無い例がある）。**入力として別途受け取る**。`__head` は **2つの `<dl>`** に分け、各 `<dl>` に `<dt>/<dd>` を2組ずつ入れる（1つ目: クライアント＋担当部署 / 2つ目: 業界＋支援内容）。
- **プロジェクト概要 = Word冒頭のリード段落群**（タイトル直下の数段落。「今回の取り組みについて…に話を伺いました。」の段落まで）。`<dd>` 内に段落ごと `<p>` で入れる。
- ロゴ `img` の `alt` は実ページで空（`""`）の例と社名テキストの例の両方がある。**既定は空 `alt=""`**、必要なら社名。§9で要確認に回す。

### 6-3. CTAブロック（定型／上下で文言が変わる／DLボタンはカテゴリで決まる）

CTAは2ボタン構成。**1つ目（左）`mod-btn-conversion`「法人向けご相談」は全カテゴリ共通**。**2つ目（右）`mod-btn-secondary` のDLボタンがカテゴリで変わる**：アカデミー・コンサルティング＝事例集（`akkodis-casestudies`）／ソリューション＝会社案内（`corporate-profile`）。

**アカデミー・コンサルティング（DL＝事例集）・上段（from=top）**：
```html
<div class="mod-lyt-cv">
<div class="mod-btnlayout">
<div class="mod-btnlayout__item">
<p class="mod-btnlayout__subText">情報収集だけでもお気軽にご相談ください</p>
<div class="mod-btn"><a href="https://www.akkodis.com/ja/contact/clients/inquiry/input?from=top" class="mod-btn-conversion mod-wide-100per_narrow" target="_blank">法人向けご相談</a></div>
</div>
<div class="mod-btnlayout__item">
<p class="mod-btnlayout__subText">課題背景〜成果までの具体事例をご紹介</p>
<div class="mod-btn"><a href="https://www.akkodis.com/ja/trends/download/akkodis-casestudies?from=top" class="mod-btn-secondary mod-wide-100per_narrow">事例集をダウンロード</a></div>
</div>
</div>
</div>
```
**アカデミー・コンサルティング・下段（from=bottom）**：上と同形で、`from=top`→`from=bottom`、subTextを `貴社の課題、一度整理しませんか` / `活用ポイントと成果を短時間で把握` に置換。

**ソリューション（DL＝会社案内）の場合**：2つ目の `__item` を差し替え（上段）。
```html
<div class="mod-btnlayout__item">
<p class="mod-btnlayout__subText">AKKODiSの事業内容・提供領域を一冊に集約</p>
<div class="mod-btn"><a href="https://www.akkodis.com/ja/trends/download/corporate-profile?from=top" class="mod-btn-secondary mod-wide-100per_narrow">会社案内をダウンロード</a></div>
</div>
```
その場合の下段は、1つ目subText `貴社の課題、一度整理しませんか`、2つ目subText `貴社課題に対応可能な領域をご確認ください`＋`corporate-profile?from=bottom`。1つ目ボタンは全カテゴリ・上下とも `法人向けご相談`（`inquiry/input?from=…`, `mod-btn-conversion`, `target="_blank"`）。

- 1つ目（ご相談）ボタンのみ `target="_blank"`。2つ目（DL）には付かない。
- 文言・href は**定型**。原稿に無くてもこの通り出力する。**2つ目のDLボタンは入力カテゴリで決まる**（アカデミー・コンサルティング＝事例集 `akkodis-casestudies`／ソリューション＝会社案内 `corporate-profile`）。カテゴリ未指定なら生成前に質問（§9・実行前確認）。※実ページのボタンは左＝黄色 `mod-btn-conversion`（ご相談）／右＝白 `mod-btn-secondary`（DL）。

### 6-4. Indexパネル（原稿に見出し＋箇条書きがある場合）

```html
<ul class="mod-panel-anchor">
<li class="mod-panel-anchor__item"><a href="#anc-01">《セクション見出し1》
<ul class="mod-panel-anchor__list">
<li>《サブ項目1》</li>
<li>《サブ項目2》</li>
<li>《サブ項目3》</li>
</ul>
</a></li>
<li class="mod-panel-anchor__item"><a href="#anc-02">《セクション見出し2》
<ul class="mod-panel-anchor__list">
<li>《サブ項目1》</li>
<li>《サブ項目2》</li>
<li>《サブ項目3》</li>
</ul>
</a></li>
<li class="mod-panel-anchor__item"><a href="#anc-03">《セクション見出し3》
<ul class="mod-panel-anchor__list">
<li>《サブ項目1》</li>
<li>《サブ項目2》</li>
<li>《サブ項目3》</li>
</ul>
</a></li>
</ul>
```

- **サブ `<ul class="mod-panel-anchor__list">` は `<a>` の内側**（見出しテキストの直後）。`mod-text-link` 等のクラスは付けない（insightsの `mod-box` とは別物）。
- 見出しテキストとサブ項目は **Word由来**（「`抱えていた課題：`＋3点」等の太字見出し＋箇条書きブロック）。生成しない。**原稿にこのブロックが無ければ Indexパネルは省く**。
- **Indexに載せるのは中核セクションのみ**（通常 課題／取り組み／成果 の3つ）。`今後の展望`・`ビジョン` 等のセクションは Index に載せない（が、本文の H2 には id を振る。§6-5）。
- `href="#anc-NN"` と本文 `<h2 id="anc-NN">` を一致させる。

### 6-5. 見出し H2 / H3

```html
<h2 id="anc-01">《セクション見出し》</h2>
<h3>《サブ見出し》</h3>
```

- **すべての本文H2に連番 `id="anc-NN"`** を上から順に振る（`anc-01`, `anc-02`, …）。Indexに載らないセクション（今後の展望 等）にも id は振る（例: 4番目のセクションなら `anc-04`）。Indexパネルが無い記事でもH2に `anc-NN` を振る。
- `<h3>` はサブ見出しがある記事のみ。id・class は付けない。**Wordで2段落に割れているH3は1つに結合**（例: `《前半――》` ＋ `《後半》` を1つの `<h3>` に）。原稿にサブ見出しが無ければ `<h3>` は省く。
- Word見出しの飾り（`■`・`●` 等の接頭辞）は**剥がす**。

### 6-6. 本文段落

**話者ラベル付きQ&A（基本構成）**：
```html
<p><strong>—《質問文》</strong></p>
<p><strong>《姓》さま：</strong> 《回答本文》</p>
<p><strong>AKKODiS 《姓》：</strong> 《回答本文》</p>
<p>《同一話者の続き段落（話者ラベル無し）》</p>
```

- 質問行：`<p><strong>—《質問》</strong></p>`（行頭ダッシュは原稿の字を保持。`—`/`―` 等）。質問のみで1段落、`：`は付けない。
- 回答：`<p><strong>《ラベル》：</strong> 《本文》</p>`。**ラベルと本文は同一 `<p>`**、`</strong>` の直後に半角スペース＋本文。コロンは全角 `：`。
- **話者ラベルの形式**（insightsの `氏名［以下、略称］：` は使わない）:
  - 顧客（非AKKODiS）= `《姓》さま：`。
  - AKKODiS = `AKKODiS 《姓》：`（`AKKODiS《姓》：` のように詰める原稿もあるが**出力は `AKKODiS 姓：`〈半角スペース入り〉に正規化**）。
- **Wordでラベルが太字でなくても、出力は必ず `<strong>` で囲む**（原稿によって太字が不安定）。
- 同一話者の回答が複数段落に渡る場合、**2段落目以降はラベル無しの裸 `<p>`**。

**三人称ナラティブ（話者ラベル無しの記事）**：
```html
<p>《地の文の段落》
<br /><br />
《次の段落》
<br /><br />
「《人物の直接引用》」
</p>
```

- 話者ラベル付きQ&Aでない記事（地の文＋「」引用）は、**話者ラベルを付けない**。実ページの慣習では**1つのH2セクションを単一 `<p>` にまとめ、段落間を `<br /><br />`** でつなぐ。「」の直接引用も同じ `<p>` 内の一段落として `<br /><br />` 区切りで入れる。`<strong>` ラベルや `<blockquote>` は使わない。導入ボックスが無い記事のリード段落（H2前）も同じ要領。
- ※段落マークアップ（別 `<p>` か `<br /><br />` か）は §2 のとおり実ページに合わせる。

### 6-7. 画像ブロック（本文の出現位置に挿入）

**キャプション有り**（人物の初出など）:
```html
<div class="mod-lyt-media mod-lyt-media--single">
<div class="mod-lyt-media__img">
<img src="/-/media/project/akkodis/akkodis/ja/images/client-stories/《slug》/img-《役割》.webp?la=ja" alt="《alt》" />
</div>
<p class="mod-lyt-media__caption">《会社》<br />《部署・肩書》<br /><strong>《氏名》</strong></p>
</div>
```

**キャプション無し**（同一人物の2枚目以降など）:
```html
<div class="mod-lyt-media mod-lyt-media--single">
<div class="mod-lyt-media__img">
<img src="/-/media/project/akkodis/akkodis/ja/images/client-stories/《slug》/img-《役割》.webp?la=ja" alt="《alt》" />
</div>
</div>
```

- **本文中の画像は基本すべて `mod-lyt-media--single`**（中央・標準幅）。確認した実ページでは左右フロートは使われていない。左右寄せ用クラスは提供ソースに無いので**作らない**（フロートが必要なレイアウトはユーザー指示を仰ぐ）。
- **キャプションの構成**: `会社`（プレーン）＜br＞`部署・肩書`（プレーン）＜br＞`氏名`（強調）。
  - **氏名の強調タグは実ソースで2通り**：`<strong>《氏名》</strong>`（基本）と `<span class="mod-lyt-media__pointText">《氏名》</span>`。**基本は `<strong>`**、実ページが `pointText` ならそれに合わせる。各1例からの確認なので§9で要確認に回す。
  - **顧客の氏名には「さま」を付ける**（例: `《姓 名》さま`）。**AKKODiS の氏名には付けない**（例: `《姓 名》`）。
  - 複数人が写る場合は氏名行に列挙（例: `<strong>《姓 名》さま、《姓 名》さま</strong>`）。会社・部署は代表で1回。
- **キャプションには2形態がある**（Wordマーカーの中身で判断）:
  - **(a) 人物プロフィール型**（上記）= 会社＜br＞部署・肩書＜br＞氏名（強調）。マーカーが人名（`【キャプション】《姓》さま` 等）のとき。
  - **(b) 説明文型** = `<p class="mod-lyt-media__caption">《説明文》</p>` の**1行のみ**（会社/部署/氏名に分けない・強調なし）。マーカーが説明文や集合の呼称（`【キャプション】AKKODiS 《姓》（写真左）、《姓》（写真右）`、`【キャプション】《集合写真の一言》` 等）のとき。AKKODiSメンバーの集合写真やシーン写真で使う。原稿マーカーの文言をそのまま入れる。
- **キャプションの有無**は Word の `《キャプション》`／`【キャプション】` マーカーに従う（マーカーのある画像＝キャプション付き）。後から追加された無マーカー画像はキャプション無し。
- キャプション本文（会社の短縮形・部署・肩書）は**編集判断**。Wordマーカーは氏名しか持たないことが多く、会社略称・部署・肩書はリード段落等から補うため、§9で要確認に回す。

## 7. 話者ラベル・段落・引用・リンク・商標の規則

- **話者ラベル（Q&A記事）**: §6-6。顧客＝`姓さま：`／AKKODiS＝`AKKODiS 姓：`。必ず `<strong>`。質問は `—質問` を `<strong>`。
- **段落マークアップ**: 基本は段落ごと別 `<p>`。実ページが `<br /><br />` 連結（ナラティブ記事に多い）ならそれに合わせる（§2, §6-6）。
- **直接引用「」（ナラティブ記事）**: `<blockquote>` を使わず、地の文と同じ `<p>` 内の段落として扱う。
- **太字/斜体**: `<strong>` / `<em>` のみ。Word由来の色span・装飾spanは捨てる。例外はキャプションの `mod-lyt-media__pointText`（§6-7）。
- **外部リンク**（出現時）: `<a rel="noopener noreferrer" href="《絶対URL》" target="_blank">《表示テキスト》</a>`。URLは `word/_rels/document.xml.rels` の `TargetMode="External"` から取得。※client-stories原稿では稀（確認した原稿では外部リンク0）。
- **CTAボタンのリンク**: §6-3の定型（`mod-btn-conversion`/`mod-btn-secondary`、ご相談ボタンのみ `target="_blank"`、`rel` は付けない＝実ソース準拠）。
- **IOWN®**（出現時）: `IOWNⓇ`/`IOWN®` は `IOWN®`（インライン、`<sup>`で囲まない）。`™`もインライン。複合語IOWNは原稿の付け方を保持。判断に迷う商標は `akkodis-proofreader`/`akkodis-brand-core` の固有名詞ルールに従い、原稿の文言を保持する（勝手に直さない）。

## 8. 画像の位置・キャプション・alt の決め方

- **位置**: Wordの `《キャプション》氏名`／`【キャプション】説明` マーカー、および埋め込み画像（`word/media/imageN`）の出現順を本文に突き合わせ、**原稿で画像が現れる位置にブロックを挿入**する。マーカー表記は揺れる（`《》`・`【】`等）ので両方拾う。
- **キャプション有無**: マーカーのある画像＝キャプション付き。マーカー無しで追加された画像＝キャプション無し（§6-7）。
- **alt**: 画像を目視し人物・所属で決める。慣習は記事により不規則（短く `会社+氏名`〈顧客はさま付き／AKKODiSは無し〉の例と、キャプションと同じ所属フル＋氏名を1行にした例の両方がある）。**alt は要確認に回す**。
- **人物特定**: MV画像で各人の顔・服装を把握してから各写真を照合。`view` で `word/media/imageN` を直接開くと鮮明。
- **配置の根拠と人物特定は §10 で1〜2行で伝える。**
